这篇主要整理 Java 的基础特点，以及常见面试里会被问到的优劣势对比。

本文适合拿来快速过一遍 **Java 基本特性、优点和缺点**。

<!-- more -->

## Java 核心特点

### 1. 平台无关性

- Java 强调“**一次编写，到处运行**”。
- 源码会先编译成字节码，再交给 JVM 执行。
- 只要目标系统安装了对应的 JVM，同一份程序通常就可以直接运行。

### 2. 面向对象

- Java 以**类**和**对象**为核心。
- 常通过**封装、继承、多态、抽象**来组织代码。
- 这种设计更利于代码复用，也方便后期维护和扩展。

### 3. 自动内存管理

- Java 内置垃圾回收机制（GC）。
- 无用对象会被自动回收，减少手动管理内存的负担。
- 这样可以降低内存泄漏和野指针一类问题的出现概率。

## Java 的优势

### 1. 跨平台兼容性强

- 依托 JVM，Java 在不同操作系统之间有很强的兼容能力。
- 这也是它长期成为企业级开发主力语言的重要原因之一。

### 2. 生态体系成熟

- Java 拥有非常完整的生态。
- 比如 Spring、Spring Boot、Hibernate 等框架都很成熟。
- 各类工具链、开源库、社区资源也比较完善。

### 3. 开发体验稳定

- 自动 GC 降低了内存管理成本。
- 原生支持多线程，适合并发编程。
- 安全模型相对成熟，比较适合服务端和网络环境开发。

### 4. 适合大型项目

- Java 版本演进通常比较重视向后兼容。
- 在大型系统、企业项目、长期维护项目中表现稳定。

## Java 的劣势

### 1. 性能存在额外开销

- 和 C++、Rust 这类原生语言相比，Java 依赖 JVM 运行时。
- 因此在启动速度、运行时开销方面通常会多一层成本。
- 在部分场景下，性能表现不如更贴近底层的语言。

### 2. 语法相对繁琐

- Java 的样板代码偏多。
- 和 Python 这类更轻量的语言相比，开发时书写成本更高。
- 对于简单需求，代码显得不够简洁。

### 3. 资源占用较高

- JVM 本身会占用一定内存。
- 在资源受限环境中，比如部分嵌入式场景，Java 并不算友好。

### 4. 灵活性相对一般

- Java 的面向对象风格比较强。
- 在一些简单业务或快速脚本类需求里，写法会显得偏重。
- 虽然也支持函数式编程，但整体自然度通常不如一些现代语言。

JVM、JRE、JDK 三者关系
三者是包含关系：JDK 包含 JRE，JRE 包含 JVM。JVM 是跑字节码的虚拟机，JRE 是跑 Java 程序的运行环境，JDK 是给开发者用的开发环境，包含了编译、调试等工具。开发用 JDK，运行用 JRE。

编译型语言：先把源码整体编译成对应平台的机器码，运行时直接执行，速度快但跨平台差，比如 C/C++、Go、Rust。
解释型语言：逐行 / 逐段解释为机器码执行，无需提前编译，跨平台好但运行稍慢，比如 Python、JavaScript、PHP。

Java 数据类型分为基本数据类型和引用数据类型两大类：
基本类型共 8 种：4 种整数（byte/short/int/long）、2 种浮点（float/double）、1 种字符（char）、1 种布尔（boolean），直接存储值，占用固定内存。
引用类型包括类、接口、数组，存储的是对象的内存地址，对象本身存放在堆内存中。

Java 数据类型转换主要分两种：
自动类型转换（隐式）：小范围类型转大范围类型，编译器自动完成，无精度损失，比如 int → long、float → double。
强制类型转换（显式）：大范围转小范围，需手动写 (目标类型)变量，可能出现精度丢失或数据溢出，比如 double → int。

对象引用转换依赖继承 / 实现关系，核心分两类：
向上转型：子类 → 父类 / 接口，自动完成，安全可靠，会隐藏子类特有方法，是多态的基础，比如 Animal a = new Dog();。
向下转型：父类 → 子类，必须手动强转，且需先用 instanceof 校验实际类型，否则会抛 ClassCastException，仅当对象真实类型是目标子类时才合法。

double 是二进制浮点类型，无法精确表示 0.1 这类十进制小数，运算时会产生微小误差（比如 0.1+0.2≠0.3），不适合金融、金额等高精度场景。
BigDecimal 是任意精度的十进制类型，能精确表示和计算小数，彻底避免精度丢失，保证财务等关键业务计算的准确性。

装箱：基本数据类型（如int）自动转换为对应的包装类对象（如Integer），是 JDK5 + 的语法糖，由编译器自动完成。拆箱：包装类对象（如Integer）自动转换回对应的基本数据类型（如int），同样由编译器自动处理。

Integer 是 int 的包装类，核心作用是弥补基本类型的面向对象缺陷：
适配对象场景：int 不是对象，无法存入集合（如 List、Map）或作为泛型参数，Integer 让 int 具备对象特性。
提供工具能力：内置 parseInt()、valueOf() 等方法，以及 MAX_VALUE 等常量，方便数值操作。
支持 null 语义：int 默认值为 0，Integer 可赋值为 null，用于表示 “值不存在”（如数据库空值）。
缓存优化：JDK 对 -128~127 的 Integer 做了缓存，减少内存开销。

面向对象三大核心特征：封装：隐藏内部实现细节，对外提供公共访问接口，保证数据安全与代码隔离。继承：子类复用父类属性与方法，建立类的层级关系，减少重复代码。多态：同一方法在不同子类有不同实现，基于方法重写与向上转型，提升代码扩展性。

多态是面向对象提高代码扩展性的核心特性，指同一方法调用，不同子类对象表现出不同行为。
它有三个必备前提：
存在继承或实现关系；
子类对父类方法进行重写；
父类引用指向子类对象（向上转型）。
多态分为两种：
编译时多态：方法重载，编译期确定调用哪个方法；
运行时多态：方法重写，JVM 运行期动态绑定实际对象的方法，这也是面试常考的核心多态。
最终价值是解耦、易扩展，符合开闭原则，不用修改原有代码就能新增功能。

Java抽象类和接口的区别
继承限制：抽象类是extends单继承，接口是implements多实现；
成员特性：抽象类可有成员变量、构造器、普通方法，接口变量默认public static final，JDK8 + 支持默认 / 静态方法；
设计思想：抽象类是is-a模板设计，接口是like-a行为规范。
